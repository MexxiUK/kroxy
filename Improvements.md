## Time-limited API keys
When generating an API key there is a dialogue box pop up. Add a setting to it that allows the user to specify a validity period. Offer options as buttons for 24 Hours (starting from the moment it's generated), 1 Week, 1 Month or Custom and allow the user to pick a date/time. Perhaps we could make it per application too?

## User Login
At the top right of the page we have the user's email. It would be preferred if the user had to specify a first name on sign up and we display their first name instead of email.

## OIDC Providers
We ought to simplify this as much as possible to guide the user through the process where possible.

## WAF Rules
We should expect the user to be using this for multiple applications and hence there should be configurable WAF rules per application.

## IP Lists
I would expect the user to be able to geofence their app to certain countries, or if they choose, block certain countries from accessing it.

## UI
Review again to make sure things appear without being squished and there is sufficient space around elements.

## Onboarding
I want you to rework the onboarding with smooth animations and multiple choice questions that will allow the app to be pre-configured to the user's requirements. Take inspiration from Apple's setup for the iPhone or Microsoft Windows but keep the app's branding and colours in mind.
